# Лабораторная работа №2. Random Forest

## Описание метода

Random Forest — ансамблевый метод, основанный на двух идеях:

1. **Bagging (Bootstrap Aggregating)** — каждое дерево обучается на bootstrap-выборке (случайная выборка с возвращением). Объекты, не попавшие в выборку, образуют OOB-множество и используются для оценки качества без отдельного валидационного набора.

2. **Метод случайных подпространств (RSM)** — при обучении каждого дерева случайно выбирается подмножество признаков. Это снижает корреляцию между деревьями и повышает устойчивость ансамбля. Обычно выбирается sqrt(p) признаков (для классификации).

Итоговое предсказание — усреднение вероятностей по всем деревьям (soft voting).

**OOB-оценка** позволяет оценить качество модели без выделения отдельной валидационной выборки: для каждого объекта предсказание строится только по тем деревьям, в обучении которых он не участвовал.

**Важность признаков OOB^j** — для j-го признака выполняется случайная перестановка его значений в OOB-выборке; важность равна падению OOB-точности.

## Датасет

Heart Disease (Statlog) из OpenML — 270 объектов, 13 признаков, бинарная классификация (наличие/отсутствие заболевания сердца). Признаки: возраст, пол, тип боли в груди, давление, холестерин, ЭКГ, пульс, наличие стенокардии и др. Разбиение: 80% обучение (216), 20% тест (54).

## Реализация

Алгоритм реализован в `source/model.py`. В качестве базовых алгоритмов используются `DecisionTreeClassifier` из sklearn. Каждое дерево обучается на bootstrap-выборке с случайным подмножеством признаков. OOB score вычисляется автоматически при обучении.

## Grid Search по OOB

Сетка параметров:
- `n_estimators`: [50, 100, 150]
- `max_features`: ['sqrt', 'log2', 0.5]
- `max_depth`: [5, 10, 15, None]
- `min_samples_split`: [2, 5]

Лучшие параметры (по OOB = 0.8519):
```
max_depth: 15, max_features: 0.5, min_samples_split: 5, n_estimators: 150
```

## Выводы

Реализация Random Forest показывает результаты, сопоставимые со sklearn. На данном датасете custom-реализация даже немного превосходит эталонную по accuracy и OOB — это объясняется небольшим размером выборки и стохастичностью bootstrap. По времени обучения custom-реализация быстрее из-за меньших накладных расходов sklearn на малых данных; на больших датасетах sklearn будет быстрее за счёт Cython-оптимизаций и параллелизма. OOB-оценка позволяет подбирать гиперпараметры без отдельной валидационной выборки, что важно при малом объёме данных.
